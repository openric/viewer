OpenRiC · Viewer demo

npm · source
Server — pick Heratio (live reference implementation), Static fixtures (service-worker replays the OpenRiC conformance pack — proves the viewer is implementation-neutral), or Custom URL for any third-party OpenRiC-conformant server.
Start — path of the root entity to draw. The dropdown lists five curated examples; type any path and hit Enter. Shape: /informationobject/<slug>, /actor/<slug>, /place/<id>, /default/recordset/<id>, etc.
Mode — 2D (cytoscape force-directed, fast) or 3D (WebGL-based, draggable camera). Switch any time and hit Load.
Hover a node to see its full identifier, RiC-O type (rico:*), and every property the server returns. Hover an edge (if available) for its predicate.
Click a node to drill into its subgraph — the Start field updates and the graph reloads around the new root.
3D controls: left-drag to orbit, right-drag to pan, scroll to zoom. Click-to-drill also orbits the camera to the selected node.
About the reference backend. This demo fetches from ric.theahg.co.za/api/ric/v1 — a public, standalone Laravel service that implements the OpenRiC spec against a real archival database. The operational GLAM platform (Heratio) is itself a consumer of this API — every mutating admin action in Heratio calls /api/ric/v1/* with an X-API-Key, same surface you're using here. No privileged shortcut.
